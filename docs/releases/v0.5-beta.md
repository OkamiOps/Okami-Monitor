# OKAMI Mission Control 0.5-beta

Release beta focado em simplificar a operação de agentes Okami e deixar o painel pronto para um fluxo mais realista de uso por pessoas técnicas e não técnicas.

> Versão do pacote: `0.5.0-beta`  
> Tag do GitHub: `v0.5-beta`

## Resumo Executivo

O `0.5-beta` consolida o Mission Control como cockpit operacional para agentes. O ponto mais importante é que **Config, Agentes e Hermes deixam de competir como conceitos separados**: a experiência principal agora fica em uma única aba **Agentes**, onde a pessoa conecta o servidor SSH, conecta agentes, cria Okami API Keys automaticamente e acessa os arquivos/configurações relevantes.

Também entram nesta entrega:

- correção das métricas de tokens por período;
- expansão de PT-BR/EN em toda a UI;
- coleta mais robusta de dados reais via Hermes/SSH;
- API Keys com hash, escopos e segredo cifrado;
- fluxo guiado para conexão automática de agentes;
- documentação e changelog completos.

## Principais Features

### 1. Aba Agentes Unificada

A navegação foi simplificada. Em vez de espalhar o usuário por Config, Agentes e Hermes, a aba **Agentes** reúne:

- acesso do painel;
- configuração SSH do servidor onde os agentes rodam;
- lista de agentes conectados;
- conexão de agente externo;
- arquivos editáveis por agente;
- instâncias;
- comandos permitidos;
- escopos e pendências;
- área avançada de keys para suporte.

O objetivo é reduzir ambiguidade: o usuário vai para **Agentes** quando quer conectar, configurar, operar ou preparar acesso.

### 2. Okami API Key Automática por Agente

O fluxo manual de "gere uma key, copie, abra arquivo, cole e salve" foi removido do caminho principal.

Agora, ao clicar **Conectar agente**, a API:

1. normaliza o cadastro do runtime/agente;
2. cria uma Okami API Key com escopos recomendados;
3. salva o segredo cifrado no cofre;
4. registra apenas prefixo, fingerprint e referência segura;
5. tenta escrever `.okami.env` no workspace remoto via SSH.

Se o SSH ainda não estiver configurado, a key fica pronta no cofre e o painel mostra o estado sem exigir edição manual do usuário.

### 3. Fluxo Mais Simples Para Usuários Leigos

O release reorganiza a experiência para evitar termos soltos e telas duplicadas:

- **Servidor dos agentes**: configura a conexão SSH geral.
- **Agentes conectados**: conecta OpenClaw, OpenHuman, Claude, Codex, agente principal e agentes externos.
- **Acesso e keys (avançado)**: só backup, suporte, colar key existente ou geração manual.

O painel passa a explicar o que está acontecendo sem pedir ao usuário que entenda backend, env file ou secret vault antes de operar.

### 4. Overview e Tokens Corrigidos

Foi corrigido o problema em que filtros de 7 dias ou 30 dias podiam mostrar menos tokens que 24 horas por inconsistência de agrupamento.

Agora:

- o card principal **Tokens** respeita o filtro ativo;
- `1h`, `24h`, `7d` e `30d` usam dados agregados coerentes;
- dias sem eventos entram como zero na janela diária;
- o card lateral de período e o card principal usam a mesma base;
- o demo vivo não parece mais um número fixo desconectado.

### 5. Kanban e Dados Reais do Hermes

O Kanban passa a agregar boards detectados no Hermes em vez de depender apenas de um board padrão.

Melhorias incluídas:

- seleção de boards;
- contagem coerente de tasks e bloqueios;
- cards com dados reais quando o backend retorna registros;
- detalhe de task com descrição, status, resultado, skills, comentários, work log e run history;
- empty states mais claros quando o `kanban.db` não retorna dados auxiliares.

### 6. Internacionalização PT-BR e EN

O painel agora tem seletor de idioma no topo, com persistência no navegador.

Cobertura EN aplicada em:

- navegação;
- cabeçalhos de seção;
- cards;
- forms;

- status dinâmicos;
- empty states;
- Apps;

- CLIs;

Foi feita uma varredura renderizada com Playwright pelas 14 abas em EN, procurando termos portugueses suspeitos, e o resultado final ficou vazio.

### 7. Backend, Segurança e Persistência

O backend recebeu melhorias para tornar o fluxo de agentes viável:

- suporte a API Key com escopos;
- validação por hash SHA-256;
- segredos cifrados com AES-256-GCM;
- storage local endurecido;
- endpoints para auth/status/bootstrap/keys;
- endpoints de SSH, comando, arquivos, cron e conexão de agente;
- proxy same-origin para produção;
- suporte local via loopback para reduzir fricção em desenvolvimento;
- allowlist de comandos;
- bloqueio de escrita fora das raízes permitidas;
- `.okami.env` remoto com permissão `600`.

### 8. Pixel Office Mais Estável

O canvas Phaser do Pixel Office foi ajustado para evitar recriações desnecessárias quando a API muda dados de agentes.

Isso reduz:

- flicker visual;
- reload de vídeo/canvas;
- perda de estado visual ao navegar;
- sensação de "refresh" constante.

## Mudanças Técnicas Relevantes

- `src/App.jsx`: reorganização grande da UI, fluxo Agentes, i18n e componentes compartilhados.
- `src/lib/apiClient.js`: client para auth, API Keys, runtimes, SSH e comandos.
- `src/lib/useMissionControl.js`: same-origin/polling/SSE mais resiliente.
- `server/index.js`: auth, scopes, endpoints protegidos, agent runtime connect e SSH actions.
- `server/store.js`: storage, secrets cifrados e API key hashes.
- `server/hermesCollector.js`: coleta real mais ampla e tolerante a schema.
- `src/data/mockMissionControl.js`: demo vivo mais completo com runtimes, agentes e dados variáveis.
- `src/PixelOfficeCanvas.jsx`: estabilidade do canvas.
- `src/styles.css`: estilos para a nova experiência de Agentes, filtros, status e painéis.

## Breaking/Behavior Changes

- A tela visível **Agentes** substitui a expectativa de telas separadas Config/Agentes/Hermes.
- `#config` continua funcionando por compatibilidade, mas o usuário vê **Agentes**.
- A geração manual de key deixa de ser o fluxo principal e fica em **Acesso e keys (avançado)**.
- Em produção, o recomendado é `same-origin` para o frontend chamar `/api`.
- O pacote usa `0.5.0-beta` internamente para SemVer, enquanto o release público é `v0.5-beta`.

## Como Testar

```bash
npm install
cp .env.example .env.local
npm run dev:all
```

Abra `http://localhost:5173`.

Fluxos recomendados:

1. Trocar idioma PT-BR/EN no topo.
2. Abrir Overview e alternar `1h`, `24h`, `7d`, `30d`.
3. Abrir **Agentes**.
4. Testar o modo demo/local sem configurar token.
5. Cadastrar um agente externo.
6. Clicar **Conectar agente**.
7. Validar que a UI não pede edição manual de arquivo.
8. Abrir Kanban, Office, Pixel, Sessions, Logs, Skills, Apps, CLIs e Docs.
